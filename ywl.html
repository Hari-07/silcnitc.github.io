USING YACC WITH LEX

Integrating LEX with YACC

A parser repetitively calls the lexical anlyzer to read tokens from the input stream. From the previous documentation, we have noted that YACC is used to generate a parser and LEX is used to generate a lexical analayzer. In terms of C code, YACC generates definition for yyparse() in y.tab.c and LEX generates definition for yylex() in lex.yy.c. We have also noted that yyparse() repetitively calls yylex() to read tokens from the input stream. Up till now, for simplicity we had written a user-defined yylex() in the YACC program to read tokens from the input stream. In this section of the document we will learn how to use the yylex() definition generated by LEX. Now, instead of a user-defined yylex(), we will use the LEX-generated yylex() definition in the YACC program to read tokens from the input stream.

This can be done by "including" the file lex.yy.c in the auxiliary funnctions section of the YACC program as shown below. "Including" lex.yy.c in the auxiliary functions section would result in all of the contents of lex.yy.c being copied to y.tab.c as YACC copies all the contents of the auxiliary functions section as such into the y.tab.c file. As a result, the definition for yylex() is also contained in y.tab.c (as lex.yy.c contains the definition of yylex). 

		/* Declarations section */
	%%
		/* Rules */
	%%
	#include "lex.yy.c"

		/* Auxiliary Functions */

LEX-YACC Communication

yyparse() attempts to parse the given input using the Contect Free Grammar (CFG) specifications in the YACC program. CFG includes non-terminals and terminals or tokens. For example a token 'PLUS' is declared in a YACC program as: 

		/* Declarations section */
	
	%token PLUS

	%%
		/* Rules */
	%%
		/* Auxiliary Functions */
