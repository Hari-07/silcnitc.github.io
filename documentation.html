Target Application Binary Interface

The Application Binary Interface of eXpOS running on the XSM architecture is given here. The functions for Intialise(), Alloc() and Free() are implemented by the eXpOS library (see eXpOS high level library interface specification). Hence an ExpL compiler is only required to generate XSM target code to redirect calls to these function to the corresponding library functions. The compiler is required to generate target assembly code for the rest of the source program and fix the code, stack and heap addresses according to the ABI specification. The compiler is also required to generate an XEXE header for the executable file generated so that the resultant target file can be loaded and executed by the eXpOS loader.
